## 💫什么是线程
> - **定义**:线程是操作系统能够进行运算调度的最小单位.它被包含在**进程**之中, 是进程中的实际运作单位
> - **简单理解**:应用软件中互相独立,可以同时运行的功能
## 🗯️什么是进程
 > - **定义**: 进程是程序的基本执行实体(可以理解为一个软件运行之后就是一个进程)

## 🎓泛型
> * **泛型基础**
> 1. 可以在编译阶段约束操作的数据类型,并进行检查
> 2. 格式:<数据类型>
> 3. 注意:泛型只能支持***引用数据类型*** / ***包装类***(因为基本数据类型不能转换为Object类)
> 4. 统一了数据类型
> 5. 指定泛型的数据类型后,传递数据时,可以传递**该类**类型或者其**子类**类型(数据具备继承性)
> * **泛型类**  
> 1. 使用场景:当一个类中,某个变量的数据类型不确定时,就可以定义带有泛型的类
> 2. `public class ArrayList<E>{...}`(创建该类的对象时,E就确定类型,E相当于是是记录数据类型的变量)
> * **泛型方法**
> 1. 方法中形参类型不确定时:1️⃣使用类名后面定义的泛型2️⃣在方法申明(修饰符后面)上定义自己的泛型(只有本方法能用)
> 2. `public<T>void show(T t){...}`
> * **泛型接口**
> 1. `public interface List<E> {...}`
> 2. 泛型接口的使用方式:1️⃣实现类给出具体类型2️⃣实现类延续泛型,创建对象时再确定
> * **泛型的继承和通配符**
> 1. 泛型不具备继承性(泛型里面写的什么类型,就必须传递什么类型),但是数据具备继承性
> 2. 泛型的通配符:`?`用来**限定类型的范围**
>       * `<? extends E>`表示可以传递E及其所有子类类型
>       * `<? super E>`表示可以传递E及其所有父类类型
## 🔎多线程
>* **应用场景**: 软件中的耗时操作, 所有的聊天软件, 所有的后台服务器  --->  提高效率

>* **实现方式**:        
>1. 继承Thread类 `public MyThread extends Thread` -->  重写run方法  -->  创建子类对象  -->  .start()开启线程   
>2. 实现Runnable接口`public class MyRun implements Runnable` -->  重写run方法  -->  创建MyRun的对象(表示多线程要执行的任务)`MyRun mr = new MyRun();`  -->  创建线程对象`Thread t1 = new Thread(mr);`  -->  开启线程`t1.start();`              
>3. (可以获取多线程运行的结果):利用Callable和Future接口(可以获取多线程运行的结果) --> 创建类MyCallable类实现Callable接口 --> 重写call(有返回值) --> 创建MyCallable对象 --> 创建FutureTask对象 --> 创建Thread类的对象,并启动

>* **常用成员方法**   
>1. `void setName (String name)`设置线程的名字,构造方法也可以设置       
>2. `static Thread currentThread ()`获取当前线程的对象     
>3. `static void sleep (long time)`让线程休眠指定的时间    
>4. `setPriority (int newPriority)`设置线程的优先级 `final int getPriority ()`获取线程的优先级        
>5. `final void setDaemon (boolean on)`设置为守护线程(当其他的非守护线程执行完之后,守护线程会陆续结束"聊天"传输文件")          
>6.  `public static void yield()`出让线程 `public final void join ()`插入线程--插入到当前线程之前
>* **线程的生命周期**    
>**新建**---*start()*--->**就绪**<---*抢CPU执行权*--->**运行**--*-run()执行完毕*--->**死亡** . (还有阻塞,等待,计时等待)     

>* **线程的安全问题**    
>1. **同步代码块**:把操作**共享数据**的代码块锁起来`synchronized (锁) {..操作共享数据的代码..}`   锁:可以用当前类的字节码文件 *** **.** class   
>2. **同步方法**:把synchronized关键字加到方法上    
>3. **lock锁**,  **死锁**
>4. **模板**:1️⃣循环-->2️⃣同步代码块--->3️⃣判断共享数据是否到了末尾(如果到了末尾)--->4️⃣如果没有到末尾(核心逻辑) 

>* **生产者和消费者(等待唤醒机制)**  
>1. **作用**:实现线程之间的协作,通过这种机制,一个线程可以先暂停执行(等待), 直到另一个线程发出特定信号(唤醒).用于协调线程之间的工作，确保线程之间的有序执行
>2. **相关方法**:必须在 synchronized 块中调用
>    1.  `wait()` 释放持有的对象锁，并进入等待状态
>    2. `notify()` 随机唤醒在该对象等待队列中等待的单个线程
>    3. `nofifyAll()`唤醒在该对象等待队列中等待的所有线程
